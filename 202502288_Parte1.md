# 1. Estructuras de Datos Eficientes

## Árboles Binarios de Búsqueda (ABB)

Un Árbol Binario de Búsqueda es una estructura de datos donde cada nodo puede tener un hijo izquierdo y un hijo derecho.

La regla principal de ordenamiento es:

* Los valores menores al nodo principal se colocan en el lado izquierdo.
* Los valores mayores se colocan en el lado derecho.

Esto permite realizar búsquedas más rápidas porque no es necesario recorrer todos los elementos.

La principal desventaja ocurre cuando los datos se insertan en orden secuencial. En ese caso, el árbol pierde su forma balanceada y se convierte prácticamente en una lista enlazada.


## Árboles AVL

Un árbol AVL es un árbol binario de búsqueda auto-balanceado.

Esto significa que el árbol intenta mantenerse equilibrado automáticamente después de insertar o eliminar nodos.

El factor de balanceo se calcula así: Factor = Altura Izquierda - Altura Derecha

Los valores permitidos normalmente son:

* -1
* 0
* 1

Si el árbol pierde el equilibrio, realiza rotaciones para balancearse nuevamente.

Gracias a esto, las operaciones de búsqueda, inserción y eliminación mantienen una complejidad de: O(log n)

# 2. Fundamentos de Web APIs

## ¿Qué es una API y cómo funciona el modelo Cliente-Servidor?

Una API es un conjunto de reglas que permite la comunicación entre aplicaciones.

El modelo Cliente-Servidor funciona de la siguiente manera:

1. El cliente envía una petición (Request).
2. El servidor recibe y procesa la información.
3. El servidor devuelve una respuesta (Response).

Esta comunicación se realiza normalmente usando el protocolo HTTP.

# Verbos HTTP

## GET

GET se utiliza para recuperar información o consultar recursos existentes.

Ejemplos:

* Obtener usuarios.
* Consultar productos.

GET no modifica datos del servidor.

Además, es un método idempotente, porque repetir la misma petición no cambia la información.


## POST

POST se utiliza para crear nuevos recursos en el servidor.

Ejemplos:

* Registrar usuarios.
* Agregar productos.

POST sí modifica los datos del servidor.

También, no es idempotente, porque repetir la misma petición puede crear registros duplicados.
